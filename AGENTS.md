# AGENTS.md - プロジェクト固有の規則

このファイルは、AIアシスタントやコントリビューターがこのプロジェクトで作業する際に従うべき規則をまとめています。

## ドキュメント方針

### 全般

- **積極的に図を使う**: プロジェクト構成の視覚的説明として重要

### README.md

- **ルートに配置**: GitHub表示のため `docs/` 内に置かない
- **簡潔に保つ**: 詳細は `docs/` へリンクで誘導

### docs/ ディレクトリ

設計についての説明を中心に、基本的なガイドをドキュメントとして置く。

### ライブラリドキュメント（docs/libs/*.md）

Doxygen コメントがヘッダにあるため、ドキュメントは以下のみ:

1. **ヘッダファイルへのリンク**
2. **設計概要**（overall な説明）
3. **最小限のコードサンプル**（5〜10行程度）

## コード規約

- **C99 標準**（`-std=c99`）
- **コメント**: Doxygen 形式（`@brief`, `@param[in/out]`, `@return`, `@pre`, `@post`, 他にも必要なものがあれば用いる）
- **前置インクリメント優先**: `++i` を使用（`i++` は後置でないと実装できない場合と、慣習的な実装の場合のみ）

## CMake 規約

- **Modern CMake**: `target_*` コマンドを使用

## テスト

- **フレームワーク**: Unity (FetchContent で自動取得)
- **配置**: 各ライブラリ/コンポーネントの `test/` ディレクトリ内（同居式）
- **命名**: `test_<ライブラリ名>.c`
